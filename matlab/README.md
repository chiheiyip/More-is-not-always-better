# MATLAB EEG Export Contract

The fusion pipeline reads EEG results from:

```text
outputs/eeg/summary/all_subjects_scene_level.csv
```

This file is generated by the standalone MATLAB/EEGLAB entry in this repository:

```matlab
addpath('matlab');
run_eeg_bandpower_from_set('E:/eeg原始文件', 'outputs/eeg');
```

`run_eeg_bandpower_from_set.m` is now a compatibility wrapper around
`matlab/eeg_bandpower_pipeline/run_eeg_bandpower_pipeline.m`. The fusion code
itself does not call EEGLAB directly; it reads the CSV exported by this MATLAB
step.

Minimum columns:

```csv
subject_id,scene_id,view_dur_s
```

Recommended columns:

```csv
subject_id,participant_id,scene_id,block_id,cycle_in_block,view_start_s,view_end_s,view_dur_s,O_alpha,O_theta,O_beta,F_theta,F_alpha,F_beta,P_theta,P_alpha,P_beta,hf_ratio_20_40Hz,rms_mean_uV,peak_to_peak_uV,nan_fraction,flat_fraction,near_boundary,segment_valid_duration
```

`scene_id` is the presentation position `1..12`, matching `participants.csv:Order` and the scene manifest generated from the questionnaire.

Subject mapping:

- `subject_id` is mapped to `participants.csv:eeg_subject_id`.
- The fusion output uses `participants.csv:participant_id` as the canonical subject key.

Timing convention:

- `scene_id` should index the EEG `view` segments generated by marker transition `7 -> 8`.
- The new exporter also parses the full marker state machine: adapt, intro, eyes-closed/open baselines, view, questionnaire, gray, and rest.
- `view_start_s` and `view_end_s`, when present, should be EEG time in seconds.
- `view_dur_s` is used for synchronization QC. If missing, the Python fusion layer tries `dur_s`, `duration_s`, or `view_end_s - view_start_s`.

QC convention:

- MATLAB exports auditable QC metrics only; formal inclusion/exclusion is applied in Python with `configs/eeg_qc.json`.
- `hf_ratio_20_40Hz`, `rms_mean_uV`, and `peak_to_peak_uV` are treated with robust, data-driven thresholds by default.
- The legacy high-frequency rule `hf_ratio_20_40Hz > 0.4` is exported as `eeg_legacy_hf_flag` for audit and sensitivity checks, not as a universal EEG standard.
- `low_gamma` and high-beta metrics are exploratory because high-frequency scalp EEG is more vulnerable to muscle artifacts.

Raw EEG input:

- One `.set` file per subject in `E:/eeg原始文件`.
- Each `.set` should have a sibling `.fdt` with the same basename when the EEGLAB dataset references external binary data.
- EEGLAB must be on the MATLAB path because the script uses `pop_loadset`.
- The `.set` files should already have filtering, re-reference, and ICA or comparable preprocessing completed when those steps are part of the study protocol.
